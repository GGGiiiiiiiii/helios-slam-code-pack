# 建图与导航 · 使用说明

> 仿真 Demo 与真机流程。原理见 [`技术说明.md`](技术说明.md)，排错见 [`典型问题.md`](典型问题.md)。

---

## 1. 两种「建图」概念

| 概念 | 含义 | 本仓库用法 |
|------|------|------------|
| **在线建图** | 开车/遥控的同时 SLAM 实时生成地图 | `sim_slam` / `sim_nav` / `mapping.launch` |
| **离线存图** | 在线建图完成后，把地图**保存成文件**供以后导航 | `map_saver_cli` → `.yaml` + `.pgm` |
| **离线用图导航** | 不建图，加载已有地图 + AMCL 定位 + Nav2 | `sim_playground_nav` / `navigation.launch` |
| **纯离线建图（bag）** | 先录 rosbag，再回放做 SLAM | 本仓库**未封装 launch**，需自录 bag（见 §5） |

日常说的「离线建图」多数指：**先在线扫一遍 → 存盘 → 以后用这张图导航**。

---

## 2. 建图需要哪些数据

SLAM（`slam_toolbox`）最少需要：

| 数据 | 话题 / TF | 仿真来源 | 真机来源 |
|------|-----------|----------|----------|
| 2D 激光 | `/scan` | Gazebo 雷达 + `laser_preprocess` | SDK 双雷达 + `laser_preprocess` |
| 里程计 | `/odom` | Gazebo `planar_move` | `/sr_amr_control/odom` 或 EKF |
| 里程计 TF | `odom → base_footprint` | Gazebo | EKF / `odom_publisher` |
| 雷达外参 TF | `base_footprint → *_laser_link` | URDF | URDF / `static_tf` |
| 仿真时钟 | `use_sim_time:=true` | 必须 | 真机用 `false` |

**不需要**预先有地图；建图过程中 slam_toolbox 发布 `map → odom`。

**建图产物**（存盘后）：

```
my_map.yaml    # 元数据：分辨率、原点、pgm 路径
my_map.pgm     # 栅格图像（灰度）
```

## 3. Launch 文件一览（主链路）

| Launch | 场景 | 地图模式 |
|--------|------|----------|
| `helios_gazebo/sim.launch.py` | 仅仿真 + 激光 + RViz | 无 |
| `helios_gazebo/sim_slam.launch.py` | 仿真 + **建图** | SLAM 在线 |
| `helios_gazebo/sim_nav.launch.py` | 仿真 + 建图 + Nav2 | SLAM 在线（边走边导航） |
| `helios_gazebo/sim_playground_nav.launch.py` | playground 场地 + **预建图** + Nav2 | AMCL |
| `helios_bringup/mapping.launch.py` | **真机建图** | SLAM 在线 |
| `helios_bringup/navigation.launch.py` | **真机导航**（需已有地图） | AMCL |

`linorobot2_*` 为本地参考包，不在本仓库发布范围内。

## 4. 仿真：建图 → 存图 → 用图导航

### 4.1 环境准备（每个新终端）

```bash
export PATH="/usr/bin:$PATH"
source /opt/ros/humble/setup.bash
cd lios/helios_linorobot_ws
colcon build --symlink-install && source install/setup.bash
```

### 4.2 在线建图（minimal 场地）

**终端 A** — 启动仿真 + SLAM：

```bash
ros2 launch helios_gazebo sim_slam.launch.py
```

**终端 B** — 遥控走一圈：

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard
```

RViz Fixed Frame 选 `map`，看到栅格逐渐变大即正常。

**终端 C** — 存图（离线地图文件）：

```bash
ros2 run nav2_map_server map_saver_cli -f ~/helios_sim_map
# 生成 ~/helios_sim_map.yaml 和 ~/helios_sim_map.pgm
```

### 4.3 在 playground 场地建图

```bash
ros2 launch helios_gazebo sim_slam.launch.py \
  world:=$(ros2 pkg prefix helios_gazebo)/share/helios_gazebo/worlds/playground.world
```

需要 `~/.gazebo/models/` 里有 `playground`、`pine_tree` 等模型。

### 4.4 用预建 playground 地图导航（最快 Demo）

仓库已自带与 playground 对齐的地图：`helios_bringup/maps/playground.yaml`。

```bash
ros2 launch helios_gazebo sim_playground_nav.launch.py
```

RViz 操作顺序：

1. 等 Nav2 就绪：`ros2 lifecycle get /bt_navigator` → `active [3]`
2. **2D Pose Estimate** 设初始位姿（AMCL 必须先设位姿才有 `map` TF）
3. **Nav2 Goal** 发目标

### 4.5 用自己存的图在 playground 导航

```bash
ros2 launch helios_gazebo sim_playground_nav.launch.py map:=~/helios_sim_map.yaml
```

### 4.6 一边建图一边导航（SLAM 模式，无需预建图）

```bash
./scripts/run_sim_nav.bash
# 或：ros2 launch helios_gazebo sim_nav.launch.py
```

RViz 直接用 **Nav2 Goal** 发目标（无需 2D Pose Estimate）。

## 5. 真机：建图与导航

```bash
# 终端 1：底盘 + 雷达
ros2 launch sr_amr_control amr_control.launch.py connect_ip:=<底盘IP> lidar:=true

# 终端 2：建图
ros2 launch helios_bringup mapping.launch.py merge:=true

# 遥控走一圈后存图
ros2 run nav2_map_server map_saver_cli -f /home/admin/maps/helios_map

# 终端 2：换导航（加载地图）
ros2 launch helios_bringup navigation.launch.py map:=/home/admin/maps/helios_map.yaml
```

详见 [`真机手册.md`](真机手册.md)。

---

## 6. 可选：从 rosbag 离线建图

本仓库未提供专用 launch。若需纯离线 SLAM，需自行录制并回放：

**录制**（建图数据采集时）：

```bash
ros2 bag record /scan /odom /tf /tf_static
```

**回放 + SLAM**（需另配 slam_toolbox 与 `use_sim_time` / 时钟同步）。

一般建议直接用 §4 在线建图 + `map_saver_cli`，更简单。
