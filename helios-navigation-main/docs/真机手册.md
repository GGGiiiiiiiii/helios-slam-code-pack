## 1. 分步联调（推荐顺序）

### 步骤 1 · 底盘 + 雷达
```bash
ros2 launch sr_amr_control amr_control.launch.py connect_ip:=192.168.71.50 lidar:=true
ros2 topic echo /sr_amr_control/front/scan --once   # 看 header.frame_id
ros2 topic hz /sr_amr_control/front/scan            # ~5Hz
```
- [ ] 确认雷达 `frame_id` 与 URDF `right_front_laser_link` / `left_behind_laser_link` 一致
- [ ] RViz 目视确认哪条对角是 front/rear（STL 无法区分，反了对调 URDF xy）

### 步骤 2 · 原生里程计
```bash
ros2 topic hz /sr_amr_control/odom                  # ~3.33Hz
```

### 步骤 3 · 头部 IMU（Orbbec，开 EKF 才需要）
```bash
ros2 launch orbbec_camera gemini_330_series.launch.py \
  enable_accel:=true enable_gyro:=true enable_sync_output_accel_gyro:=true \
  enable_color:=false enable_depth:=true enable_ir:=false
ros2 topic type /camera/gyro_accel/sample           # 确认为 sensor_msgs/Imu
```
- [ ] 确认消息类型
- [ ] 补 `base_footprint → … → camera/imu` 外参（接整机 URDF 或静态 TF），否则 EKF 姿态会偏

### 步骤 4 · 桥接层 bringup
```bash
# A. 原生 odom + EKF（推荐，需先完成步骤 3）
ros2 launch helios_bringup bringup.launch.py use_ekf:=true use_odom:=false merge:=true

# B. 无 IMU 回退：速度积分里程计
ros2 launch helios_bringup bringup.launch.py use_ekf:=false use_odom:=true merge:=true
```
- [ ] `ros2 run tf2_tools view_frames` → `map→odom→base_footprint→base_link→雷达`
- [ ] `/scan` 有数据、车身四周无假障碍（自过滤参数按真实 footprint）
- [ ] `odom→base_footprint` 唯一发布者（EKF 与 odom_publisher 不同时发 TF）

### 步骤 5 · 建图 → 存图
```bash
ros2 launch helios_bringup mapping.launch.py
# 遥控走一圈后：
ros2 run nav2_map_server map_saver_cli -f /home/admin/maps/helios_map
```

### 步骤 6 · 定位 + 导航
```bash
ros2 launch helios_bringup navigation.launch.py map:=/home/admin/maps/helios_map.yaml
```
- [ ] RViz 2D Pose Estimate 设初始位姿，AMCL 收敛
- [ ] Nav2 Goal 发目标，能规划 + 避障 + 平移到点
- [ ] 导航时**双臂回收**（footprint 不含臂展）

## 2. 首次真机前置清单
- [ ] Orin 上 `colcon build`（source SDK 后）
- [ ] `sr_amr_interfaces` 在工作空间可见
- [ ] `lidar_enabled` 服务已开（或 bringup 后续自动调）
- [ ] 底盘手动模式 / 远程控制可用（`cmd_vel_relay` 会调 `remote_control_enabled`）
- [ ] 各终端 `ROS_DOMAIN_ID` 一致
