```bash

ros2 launch helios_description display.launch.py   # 只看模型/TF

```

### playground 场地（linorobot2 预建地图）

比 `minimal.world` 障碍更丰富；地图来自 linorobot2 `playground.yaml`，与场地匹配。

```bash
# 一键：playground 3D 场地 + AMCL 定位 + Nav2（默认已带好 world/map）
ros2 launch helios_gazebo sim_playground_nav.launch.py

# 仅换 3D 场地、仍用 SLAM 在线建图
ros2 launch helios_gazebo sim_nav.launch.py \
  world:=$(ros2 pkg prefix helios_gazebo)/share/helios_gazebo/worlds/playground.world
```

playground 依赖 `~/.gazebo/models/` 里的 `playground`、`pine_tree` 等模型（本机已有则无需额外下载）。

playground 流程 RViz：
1. 等 Nav2 就绪（`ros2 lifecycle get /bt_navigator` → active）
2. **必须先 2D Pose Estimate** 设初始位姿（AMCL 模式在设位姿前没有 `map` TF，Fixed Frame 报红属正常）
3. 设好位姿后 Fixed Frame 选 `map`，再 **Nav2 Goal** 发目标

遥控 / 存图：

```bash
ros2 run teleop_twist_keyboard teleop_twist_keyboard
ros2 run nav2_map_server map_saver_cli -f ~/helios_sim_map
```

## 3. 就绪标志与验收

```bash
ros2 lifecycle get /bt_navigator     # active [3] = Nav2 就绪
ros2 topic hz /scan                  # 有数据
```

RViz 里：
1. Fixed Frame = `map`；能看到紫色全局路径 `/plan`。
2. 工具栏 **Nav2 Goal / 2D Goal Pose** 发目标（不是 2D Pose Estimate）。
3. 车**平移**到点（验证 Omni 生效，非只原地转）；车身四周无假障碍圈。
4. 导航途中才有局部路径 `/transformed_global_plan`、MPPI `/trajectories`。
